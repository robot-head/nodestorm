# nodestorm v0.3 ‚Äî user graph editing, scale & nav UX, export polish

Approved 2026-07-17. Scope = the three "Next ‚Äî v0.3" clusters in ROADMAP.md,
in one release, stacked on the v0.2 branch (PR based on
`claude/feature-roadmap-1fe720`, retargets to `main` when PR #3 merges).

## 1. Ownership model

`Node` and `Edge` gain `origin: Origin` ‚Äî `enum Origin { Agent (default),
User }`, `snake_case`, `#[serde(default, skip_serializing_if = is_agent)]`
so agent-facing JSON is unchanged for agent-authored elements.

- **Server normalizes**: everything arriving via `propose_graph` /
  `update_graph` upserts is forced `origin: Agent` regardless of what the
  agent sent ‚Äî the agent cannot claim user authorship.
- **`apply_propose` preserve pass**: merge as today, then re-append
  user-origin nodes and user-origin edges from the previous doc that the
  proposal did not mention (matched by node id / edge key). A user edge
  whose endpoint no longer exists is dropped, with a warning appended to the
  tool-result warnings.
- **Adoption**: if the agent upserts a node id that was user-origin, the
  node becomes `origin: Agent` (the merge keeps user-owned position/notes/
  decided-choices as always). From then on the agent must carry it forward
  in proposes, like any of its nodes. Documented in SKILL.md.
- `update_graph`'s `remove_node`/`remove_edge` may target user-origin
  elements (explicit agent action, e.g. honoring a `removal_requested`).

## 2. Editing operations + decision events

New `DecisionKind` variants (snake_case wire tags; the payload fields are
`node_kind`/`edge_kind` because the event envelope's internal tag already
owns the name `kind`):
`node_added { node_id, label, node_kind }`,
`node_edited { node_id, label, node_kind, description }`,
`node_deleted { node_id }`,
`removal_requested { node_id }`,
`edge_added { from, to, edge_kind }`,
`edge_deleted { from, to, edge_kind }`.

Store mutations (each: mutate ‚Üí event ‚Üí activity entry; **no autoflush** ‚Äî
editing events ride along with the next Send/flush, like notes):

- `add_user_node(label, kind, position: Option<Point>) -> NodeId` ‚Äî id =
  slugified label, deduped with `-2`, `-3`‚Ä¶; `origin: User`,
  `status: Proposed`; position = the view-center point the UI passes
  (pinned), `None` falls back to auto-layout.
- `edit_node(id, label, kind, description)` ‚Äî allowed on **any** node; on
  agent-origin nodes it is a user correction the event tells the agent to
  carry forward (next agent upsert may overwrite; the event is the contract).
- `delete_node(id)` ‚Äî the approved matrix:
  - user-origin ‚Üí hard delete node + its incident edges ‚Üí `node_deleted`;
  - agent-origin ‚Üí set `status: Removed` (red dashed card) ‚Üí
    `removal_requested`; the agent applies the real removal (or pushes
    back) via `update_graph`. Idempotent: deleting an already-`Removed`
    agent node is a no-op.
- `add_user_edge(from, to, kind)` ‚Äî default `DependsOn`; rejects dangling
  endpoints, self-loops, and duplicate `(from, to, kind)` keys;
  `origin: User`, `status: Proposed` ‚Üí `edge_added`.
- `delete_edge(from, to, kind)` ‚Äî **all edge deletes are hard** (edges
  carry no choices/notes; the agent re-adds if it disagrees) ‚Üí
  `edge_deleted`.

## 3. Editing UI (buttons AND gestures)

Buttons/panel (every action a named, UIA-clickable control ‚Äî the E2E
harness drives editing through these):

- Topbar **‚Äú+ Component‚Äù**: creates at view center, selects it, panel opens
  with the edit form ready.
- Panel for the selected node adds: **Edit** form (label input, kind
  select, description textarea, Save), **Delete** (tooltip states the
  soft/hard behavior), **Connect ‚Üí** (arms connect mode; clicking a target
  card creates the edge; Esc cancels), and a **Connections** list of
  incident edges each with a ‚úï delete button.

Gestures on top:

- Double-click empty canvas ‚Üí new node at that plane point.
- Drag from a card's connect handle (‚óâ, visible on hover/selection) ‚Üí
  rubber-band line ‚Üí release over a card ‚Üí edge.
- Right-click card / background / edge ‚Üí HTML context menu (not native):
  card = Rename¬∑Connect¬∑Delete; background = ‚ÄúAdd component here‚Äù; edge =
  Delete edge.
- `Del` deletes the selection; `Esc` cancels connect mode / closes menu.

## 4. Scale & navigation

- **Search** (topbar input): live case-insensitive substring match on
  label/id/group; hits highlighted, non-hits dimmed; `Enter` cycles matches
  and zooms to each; `Esc` clears; `/` focuses the box.
- **Minimap** (bottom-right overlay next to the fit button): scaled node
  rects colored by status + viewport rectangle; click/drag pans the main
  view. Pure render from `Layout` + `ViewTransform`.
- **Keyboard** (canvas viewport, `tabindex=0`): arrows = spatial nearest-
  node selection move; `Tab`/`Shift-Tab` = document-order cycle; `Enter` =
  open panel for selection; `Del`; `+`/`-` zoom; `0` = fit; `n` = new node.
- **Zoom-to-node**: double-click a card (background double-click creates,
  so no collision); search-`Enter` also zooms.

## 5. Export polish

Topbar Export becomes **‚ÄúExport ‚ñæ‚Äù** (HTML dropdown, automatable):

1. *Export* ‚Äî unchanged v0.2 behavior (session-adjacent write, feed
   receipt) ‚Äî stays E2E-tested.
2. *Export As‚Ä¶* ‚Äî native save dialog via new dep `rfd` (sync call in the
   handler; default filename `{title-slug}-decisions.md`). Manual
   verification only (native dialogs can't be driven window-targeted).
3. *Copy Markdown* / 4. *Copy Mermaid* ‚Äî `navigator.clipboard.writeText`
   via `document::eval` (no new dep). Manual verification only ‚Äî the
   automated probes must not clobber the user's live clipboard.
5. *Export Mermaid only* ‚Äî writes `session.mermaid.md` next to the session
   file.

MCP: `export_markdown` gains optional `format: "markdown" | "mermaid"`
(serde default markdown ‚Äî backward compatible; README tools table updated).
