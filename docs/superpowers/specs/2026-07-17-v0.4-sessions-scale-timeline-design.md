# nodestorm v0.4 — concurrent sessions, >100-node scaling, session timeline

Approved 2026-07-17. All three remaining roadmap clusters, with **concurrent
per-session stores** as the multi-session model. Branch
`claude/v0.4-sessions-scale-timeline`, PR to `main` (carries the rebased
v0.3 commits — PR #4 merged into the roadmap branch, not main).

## 1. Sessions: first-class concurrent stores

New module `src/sessions.rs`:

```rust
pub struct Sessions {
    inner: Mutex<Inner>,          // { map: BTreeMap<String, Entry>, active: String }
    dir: PathBuf,                 // <data-dir>/sessions (or --sessions-dir)
    generation: watch::Sender<u64>, // bumped on list/active changes (UI resubscribe signal)
}
struct Entry { store: Arc<Store>, path: PathBuf }
```

- API: `open(dir, pinned: Option<PathBuf>) -> Arc<Sessions>` (loads every
  `*.json` in dir; migrates a legacy sibling `session.json` to
  `sessions/default.json`; guarantees ≥1 session, creating `default` empty);
  `create(name) -> Result<String>` (slugified, deduped `-2`…);
  `resolve(Option<&str>) -> Result<Arc<Store>>` (None → active);
  `get(name)`, `list() -> Vec<SessionInfo>`, `active_name()`,
  `active_store()`, `switch(name) -> Result<()>`,
  `archive(name) -> Result<()>` (moves file to `sessions/archive/`, drops
  from map, never archives the last live session; archiving the active one
  switches to another first), `save_all()`, `subscribe_generation()`.
- `SessionInfo { name, active, node_count, open_choices, agent_waiting, undelivered }`.
- Autosave: one task per session (existing `persist::autosave_task`),
  spawned on open/create; `save_all()` on shutdown.
- `--session <file>` pins that exact file as a session named after the file
  stem and makes it active (keeps the E2E harness contract; its export
  paths still derive from that file). `--sessions-dir <dir>` overrides the
  default directory.
- Only the **user** switches the active session (UI); agents address
  sessions by name.

## 2. MCP: per-session routing

- Every existing tool's params gain
  `#[serde(default)] session: Option<String>` — `None` = active session, so
  v0.3 agents work unchanged. Unknown name → invalid-params error listing
  available sessions — except `propose_graph`, which **auto-creates** the
  named session (agents can spin up parallel brainstorms).
- `await_decisions` binds to the resolved store at call time: an agent can
  block on session A while the user works in B. No cancel-on-switch — the
  whole point of per-session stores.
- New `list_sessions` tool → `{ sessions: [SessionInfo…] }`.
- `get_state` result gains `session: String` (which session it described).
- ServerInfo instructions mention sessions + list_sessions.

## 3. UI: active-session plumbing + switcher

- Context switches from `Arc<Store>` to `Arc<Sessions>`; `use_store()`
  returns the active store. The store→UI bridge selects over (active
  store's revision watch, sessions generation watch) and re-snapshots +
  re-subscribes when the generation changes. Switching resets selection,
  connect mode, search, and the open panel.
- **Sessions ▾** topbar menu (same dropdown pattern as Export ▾): one row
  per session — name, open-decisions badge, ● agent-waiting badge — click
  to switch; an inline name input + **Create** button; **Archive current**.

## 4. Scaling to >100 nodes

- **Viewport culling** (always on): only node cards whose layout rect
  intersects the view rect (±one screen of margin) render; edges render
  only if either endpoint's rect intersects. Pure helper
  `visible_set(layout, transform, view) -> (nodes, edges)` unit-tested.
- **Collapsible groups (clustering)**: `SessionState.collapsed_groups:
  Vec<String>` (serde default — persisted per session, *not* in the doc, so
  agents never see it). `layout::compute(doc, collapsed)` treats each
  collapsed group as one synthetic cluster node (id `group:<name>`) sized
  by member count; member nodes get no rects. Collapse/expand: click the
  group pill on a card / a "Collapse group" context-menu item; a collapsed
  cluster card shows the group name + member count and expands on
  double-click (or the same pill).
- **Aggregated cluster edges (the bundling story)**: edges whose endpoint
  is hidden inside a collapsed group re-route to the cluster node; parallel
  edges between the same visible pair merge into one `EdgePath` with
  `bundle_count: usize` (>1 renders thicker with a `×N` label). Delivered
  structurally instead of curve-FDEB cosmetics.
- Fixtures: `demo::big_doc(n)` (grouped grid, deterministic) and a
  `--demo-big <N>` CLI flag for manual/perf runs.

## 5. Session timeline

- **Timeline panel**: toggleable (topbar "Timeline" button) panel listing
  the persisted `decision_log` chronologically — option picks (labels
  resolved best-effort through the doc, raw ids as fallback), dismissals
  with reasons, notes, user edits (`node_added`/`node_edited`/…), and
  `flush_requested` comments — each with a `HH:MM` timestamp.
- **Export**: `render_markdown` gains a final `## Session log` section with
  one line per event (`- 2026-07-17 14:02 — picked “CRDTs” for Sync
  Engine`, `- … — comment: “looks good, proceed”`). Flush comments finally
  land in the record (closes the v0.2 scope-out).
