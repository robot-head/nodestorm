# nodestorm v0.5 + v0.6 — session management & diffing; undo/redo & bundling

Approved 2026-07-17. Both releases on `claude/v0.5-v0.6`, one PR to `main`,
per-release version-bump commits.

## v0.5

### Session lifecycle (user-only; `src/sessions.rs`)

- `rename(old, new) -> Result<String>`: slugified + deduped; save, rename
  the file, re-key the map, follow `active`, bump generation. In-flight
  awaits keep working (they hold the `Arc<Store>`); tool calls using the
  stale name get the normal unknown-session error naming what exists.
- `delete(name) -> Result<()>`: hard — remove from map and delete the file.
  Same guardrails as archive: never the last session; deleting the active
  one switches to a survivor first.
- `list_archived() -> Vec<String>` (sorted stems of `archive/*.json`) and
  `unarchive(name) -> Result<String>` (file moves back, name deduped
  against live sessions, store loaded, generation bump).
- No MCP surface for lifecycle (agents still only auto-create via propose).

### Cross-brainstorm diffing

- New pure module `src/diff.rs`:
  `pub fn diff_docs(a_name: &str, a: &SessionDoc, b_name: &str, b: &SessionDoc) -> String`
  — deterministic Markdown: `# Diff: a → b`; **Components** added / removed
  / changed (field-level: label, kind, status, group, description);
  **Edges** added / removed (by `(from, to, kind)` key); **Decisions**
  newly decided, decided differently, reopened, newly dismissed; exactly
  `_No differences._` when equal.
- MCP tool `diff_sessions { a, b }` → one plain-text block (both names
  resolved through the session manager; unknown → the usual error).
- UI: every non-active row in the switcher gets a **Compare** affordance
  that opens a right-panel (`DiffPanel`, Timeline-style) showing
  `diff_docs(active, that session)`.

### Switcher "Manage" section (active session)

Rename input + **Rename** button; **Delete current** (danger, hard);
**Archived** list with `↩ name` unarchive rows.

## v0.6

### Undo/redo (snapshot-based, honest boundaries)

- `Snapshot { label, doc, log_len }`; transient (`#[serde(skip)]`) capped
  stacks (50) on `SessionState`.
- Every user mutation checkpoints before mutating (select, dismiss, note,
  add/edit/delete node, add/delete edge). Drags checkpoint **once** via an
  explicit `checkpoint_position(node)` called at drag start; `set_position`
  itself never does.
- `undo()` / `redo()`: restore `doc` and truncate `decision_log` to the
  snapshot's `log_len` (only ever the undelivered tail), swap onto the
  other stack, activity receipt (`undid: picked …`).
- **Clearing rules** (the honesty core): any flush (manual or auto) clears
  both stacks — delivered decisions are facts; any agent mutation
  (`apply_propose`, `apply_update`, `clear_session`) clears both — undo
  never clobbers agent work. The undo window is "since the agent's last
  turn". `delivery_cursor` safety: a checkpoint's `log_len` is always ≥ the
  cursor because flushes clear first.
- `UiMeta { undo_available, redo_available }`; topbar **Undo**/**Redo**
  buttons (disabled accordingly) + `Ctrl+Z` / `Ctrl+Y` on the canvas.

### Cosmetic curve bundling (expanded graphs)

Edges spanning >1 rank are channel-routed: their bezier control points pull
into a shared lane per `(from_rank, to_rank)` group (lane index = position
within the group, document order), so parallel long edges run together
through the gutters instead of criss-crossing. Pure post-pass in
`route_edges` (ranks are already computed); deterministic; verified by a
layout unit test (long-edge paths change, short edges byte-identical) and a
`--demo-big` screenshot.
