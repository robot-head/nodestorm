# nodestorm v0.5 + v0.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** v0.5 (session rename/delete/unarchive + structured session diffing) then v0.6 (snapshot undo/redo + cosmetic curve bundling), per `docs/superpowers/specs/2026-07-17-v0.5-v0.6-design.md`.

**Architecture:** lifecycle verbs extend `Sessions` under the existing mutex+generation model; diffing is a new pure module like `export.rs`; undo is transient snapshot stacks inside `SessionState` with clear-on-flush/agent-turn rules enforced in `Store`; bundling is a `route_edges` post-pass using the ranks already computed.

**Tech Stack:** unchanged.

## Global Constraints

- Per task: full `cargo test`, `clippy --all-targets -- -D warnings`, `fmt` clean, commit. `set -o pipefail` on every gate chain.
- TDD for sessions/diff/store/layout; UI wiring covered by E2E additions.
- Determinism everywhere; store stays SDK-agnostic; lifecycle stays user-only.
