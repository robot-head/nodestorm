# v0.8 Storm UI Makeover Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restyle nodestorm with the approved "Storm" identity and make the top bar impossible to overflow at narrow window widths.

**Architecture:** Pure-presentation change plus two small behaviors (compose popover, copy-command pod). All new colors derive from existing semantic tokens via `color-mix`, so the 12 palette families × light/dark keep working untouched. Responsiveness is CSS container queries on the header — no Rust width tracking. E2E stability comes from pinning today's UIA accessible names with `aria-label`.

**Tech Stack:** Rust + Dioxus 0.7 (desktop WebView), plain CSS in `assets/main.css` (injected via `include_str!`), Windows UIA E2E in PowerShell.

**Spec:** `docs/superpowers/specs/2026-07-17-ui-makeover-storm-design.md`

**One refinement vs. the spec** (same visible outcome, less risk): the card's 3px status rail is a `.node-card::before` absolute overlay, not a flow `div.node-rail`. Cards keep their exact current heights, so `layout::estimate_height` needs **no constant change** (the spec's geometry-sync constraint is satisfied with zero churn), and cluster cards inherit the rail for free. Horizontal padding still goes 12→13px to compensate the removed 3px left border, keeping text content width at exactly 232px.

## Global Constraints

- Never edit the `[data-theme="…"]` palette blocks in `assets/main.css` (theme.rs registry tests pin them); new colors only via `color-mix(in srgb, var(--accent) N%, transparent)` etc.
- Every control whose visible text changes carries `aria_label` with today's exact UIA name — see the pinning table in the spec §2. New-name controls: only `More` (⋯ pod) and `Message to agent` (compose pod).
- `assets/main.css:996` geometry comment and `src/layout.rs:130` comment must describe reality after every card change; text content width stays 232px (wrap estimates depend on it).
- Desktop app works offline: fonts embedded as data URIs, no runtime fetches.
- Exported records unchanged (`src/export.rs` untouched).
- After every task: `cargo fmt`, `cargo clippy --all-targets -- -D warnings` clean, `cargo test` green.
- Windows E2E: `powershell -File scripts\verify-windows.ps1 -DemoShot` (fast render check) per task where noted; full `verify-windows.ps1` in Tasks 5, 6, 8.
- Commit after every task (bodies below); branch is `claude/ui-redesign-button-overflow-f1beb5`.
