# nodestorm v0.7 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** v0.7 — twelve terminal-theme families as color palettes (dark + light each), `auto | dark | light` mode with OS auto-follow, TopBar picker, global persisted preference, native title bar synced to the mode, per `docs/superpowers/specs/2026-07-17-v0.7-theming-design.md`.

**Architecture:** pure-CSS theming via `light-dark()` under `[data-theme]`/`[data-mode]` attributes on `div.app` (guarded by `@supports`); a pure `theme.rs` registry whose colocated tests `include_str!` the stylesheet and keep it in lock-step; a global `prefs.rs` preferences file (atomic writes, corrupt-safe) outside `SessionState`; `--prefs` CLI override; tao `set_theme` for the window chrome.

**Tech Stack:** unchanged.

- Per task: full `cargo test`, `clippy --all-targets -- -D warnings`, `fmt` clean, commit.

- The 16 existing `:root` token values stay byte-identical (they are the no-support fallback and the export palette's mirror).
- Theme preference never touches `SessionState`, undo, MCP, or exports.

## Self-review

- Solarized keeps its cross-background accents un-split (single values) — intentional, that is Solarized's design. Gruvbox's `--status-existing` likewise. The integrity test therefore requires `light-dark(` only on the 7 structural bg/text tokens (which can never be shared between variants) and token *presence* for all 20.
- E2E brightness thresholds chosen far from both distributions (dark ≈ 0.1, gruvbox-light ≈ 0.85).
