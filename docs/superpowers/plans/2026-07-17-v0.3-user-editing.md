# nodestorm v0.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** User graph editing (origin-tracked, event-driven), scale & navigation UX (search, minimap, keyboard), and export polish (Export menu: Save As, clipboard, mermaid-only), per `docs/superpowers/specs/2026-07-17-v0.3-user-editing-scale-export-design.md`.

**Architecture:** `Origin` field on Node/Edge with server-side normalization and a preserve pass in `apply_propose`; six new `DecisionKind` variants flowing through the existing decision log (no autoflush); all UI actions exist as named UIA-clickable controls first, gestures layered on top; export gains a dropdown menu and a `format` param on the MCP tool.

**Tech Stack:** Rust edition 2024, Dioxus 0.7 desktop, rmcp 2.2, new dep `rfd` (native save dialog only).

## Global Constraints

- `cargo clippy --all-targets -- -D warnings` and `cargo fmt --check` clean after every task; `unsafe_code = "deny"`.
- TDD: failing test first for every store/model/tool behavior; UI tasks are covered by the Task 12 E2E script additions.
- Determinism: no `HashMap` iteration on any output path; document order everywhere.
- Agent-inbound serde types keep `deny_unknown_fields`; outbound event types do not.
- Store stays SDK-agnostic (no rmcp/Dioxus imports); all rmcp contact stays in `src/server/`.
- Commit after each task with a one-line conventional message.
