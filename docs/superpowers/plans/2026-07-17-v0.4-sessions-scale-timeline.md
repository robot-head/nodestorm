# nodestorm v0.4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Concurrent named sessions (per-session stores, MCP `session` routing, UI switcher), >100-node scaling (viewport culling, collapsible groups with aggregated cluster edges), and the session timeline (panel + `## Session log` export section), per `docs/superpowers/specs/2026-07-17-v0.4-sessions-scale-timeline-design.md`.

**Architecture:** A `Sessions` manager owns `name → Arc<Store>` + the active name; the `Store` is untouched. MCP tools resolve an optional `session` param (default active; propose auto-creates). The UI re-subscribes on a generation watch. Layout gains a `collapsed` parameter producing synthetic cluster nodes and bundle-counted edges; the canvas culls to the viewport.

**Tech Stack:** unchanged (Rust 2024, Dioxus 0.7, rmcp 2.2, rfd).

## Global Constraints

- `cargo clippy --all-targets -- -D warnings`, `cargo fmt --check`, full `cargo test` green after every task; commit per task.
- TDD: failing test first for sessions/layout/export/MCP behaviors; UI wiring covered by the Task S7 E2E additions.
- Determinism everywhere (BTreeMap or sorted Vec for session listings; no HashMap iteration on output paths).
- Store stays SDK-agnostic; all rmcp contact in `src/server/`; `deny_unknown_fields` on agent-inbound params.

### Task S1: `Sessions` manager

**Files:** Create `src/sessions.rs`; modify `src/lib.rs` (+`pub mod sessions;`), `src/persist.rs` (make `load`/`save` reusable — already pub).

**Interfaces (Produces):**

```rust
pub struct SessionInfo { pub name: String, pub active: bool, pub node_count: usize,
    pub open_choices: usize, pub agent_waiting: bool, pub undelivered: usize }

impl Sessions {
    pub fn open(dir: PathBuf, pinned: Option<PathBuf>) -> anyhow::Result<Arc<Self>>;
    pub fn create(&self, name: &str) -> anyhow::Result<String>;      // slugified, deduped -2…
    pub fn resolve(&self, name: Option<&str>) -> Result<Arc<Store>, String>; // Err = "unknown session `x`; available: a, b"
    pub fn get(&self, name: &str) -> Option<Arc<Store>>;
    pub fn get_or_create(&self, name: &str) -> anyhow::Result<(String, Arc<Store>)>; // propose auto-create
    pub fn list(&self) -> Vec<SessionInfo>;                          // sorted by name, active flagged
    pub fn active_name(&self) -> String;
    pub fn active_store(&self) -> Arc<Store>;
    pub fn switch(&self, name: &str) -> anyhow::Result<()>;          // bumps generation
    pub fn archive(&self, name: &str) -> anyhow::Result<()>;         // move file → archive/, never the last; active → switch first
    pub fn save_all(&self);
    pub fn subscribe_generation(&self) -> watch::Receiver<u64>;
    pub fn spawn_autosaves(&self, handle: &tokio::runtime::Handle);  // one persist::autosave_task per session (called once; create() late-spawns via stored handle)
}
```

`open`: create dir; if dir has no `*.json` and legacy `dir.parent()/session.json` exists → move it to `dir/default.json`; load every `*.json` (stem = name) via `persist::load`; if empty create `default` (empty doc). `pinned`: load/create that file, name = file stem (slugified), insert, make active. Store the runtime `Handle` after `spawn_autosaves` so `create()` can spawn for new sessions.
